# 公开数据集训练方案

> 目标：为本系统补充大量公开数据，先用公开数据预训练/验证，再用本项目 Web 采集页采集少量本地样本微调。系统线上推理仍使用 63 维 MediaPipe 手部关键点，因此公开图片/视频数据需要先离线跑 MediaPipe，把图像帧转换为与线上一致的关键点特征。

## 推荐优先级

| 优先级 | 数据集 | 适合任务 | 规模 | 用法 |
| --- | --- | --- | --- | --- |
| 1 | HaGRID / HaGRIDv2 | 静态手势 | HaGRID Kaggle 版约 552,992 张 FullHD RGB 图像，18 类，约 716GB | 训练静态分类器，优先映射 palm、fist、thumbs_up、ok 等类别 |
| 2 | 20BN-Jester | 动态手势 | Qualcomm 页面说明共 148,092 个视频目录，下载约 22.8GB，25 类 | 训练 swipe、zoom、none 等动态手势 |
| 3 | EgoGesture | 动态/静态混合 | 官方说明含 2,081 个 RGB-D 视频、24,161 个手势样本、2,953,224 帧、50 人、83 类 | 做动态手势泛化训练和论文对比实验 |
| 4 | SHREC'17 / DHG-14/28 | 骨架动态手势 | 常用基准为 2,800 条序列、14/28 类、22 个手部关节 | 做 LSTM/时序模型的骨架基线，适合作论文实验 |
| 5 | NVGesture | 动态手势 | 常见镜像说明约 1,532 个动态手势、25 类，RGB/Depth/IR | 适合车载/非接触动态手势扩展 |

## 下载方式

这些数据集体量大，并且常需要同意许可或登录账号。建议不要把原始数据提交到 Git；项目 `.gitignore` 已忽略 `data/`。

```powershell
# HaGRID Kaggle，需先配置 kaggle.json
python backend/scripts/download_public_datasets.py --dataset hagrid

# Jester Kaggle 镜像或官方下载后的整理流程
python backend/scripts/download_public_datasets.py --dataset jester

# 只打印所有数据集入口
python backend/scripts/download_public_datasets.py --dataset all --dry-run
```

## 预处理流程

1. 下载原始图片/视频到 `data/raw/<dataset>/`。
2. 对每张图片或视频抽帧运行 MediaPipe Hands。
3. 检测到 21 个关键点后，执行与前端完全一致的归一化：以手腕为原点，以手腕到中指根距离为尺度。
4. 静态样本保存为 `label + 63维特征`，写入 PostgreSQL 或 CSV。
5. 动态样本统一采样/插值为 30 帧，保存为 `label + 30x63特征`。
6. 运行 `backend/scripts/train_static.py` 和 `backend/scripts/train_dynamic.py` 训练模型。

## 当前建议

先做小规模可控闭环：

1. HaGRID 每类抽样 2,000 到 5,000 张，训练静态模型。
2. Jester/EgoGesture 每个动态类抽样 1,000 到 3,000 段，训练动态模型。
3. 用项目采集页补采本地摄像头样本，每类至少 200 到 500 条，用于微调和毕业设计演示。

这样训练规模足够支撑论文实验，又不会一开始被数百 GB 数据阻塞。
